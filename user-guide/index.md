The Australian Nextflow Seqera Service provides a central web platform for managing, launching and monitoring Nextflow workflows on connected compute infrastructure.

This guide is organised around what you want to do. Most researchers can begin with **Access and accounts** and then move directly to **Run your first workflow**. The infrastructure configuration sections are primarily for workspace administrators and technical staff.

## Choose your task

### I want to access the service

Start with [Access the service](/user-guide/service-access) to join an existing workspace or request a new organisation workspace. Then follow [Log in and switch workspaces](/user-guide/log-in).

### I want to run a workflow

Go to [Run your first workflow](/user-guide/run-first-workflow) for the full user journey: choose a workflow, provide inputs, select compute, launch the run and find the outputs.

### I administer a workspace

See the [Workspace administrator overview](/user-guide/workspace-administration) for participant management, shared resources, credentials and compute responsibilities.

### I need to connect an HPC or cloud environment

Begin with [Configure compute environments](/user-guide/compute-env), then use the platform-specific guidance for [HPC and Tower Agent](/user-guide/hpc-agent) or [cloud and other infrastructures](/user-guide/cloud-compute).

## Key concepts

- **Organisation:** A top-level Seqera structure used to group workspaces and members.
- **Workspace:** A shared area containing workflows, compute environments, credentials, runs and participants.
- **Workflow or pipeline:** A Nextflow workflow that can be configured and launched through Seqera.
- **Compute environment:** The HPC, cloud or other execution platform where workflow tasks run.
- **Credentials:** Secure connection information used by Seqera to access a compute environment.
